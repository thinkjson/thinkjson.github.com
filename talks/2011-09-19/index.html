First, an introduction

Saiku is a modular open-source analysis suite offering lightweight OLAP which remains easily embeddable, extendable and configurable.

Our RESTful server connects to existing OLAP systems, which then powers user-friendly, intuitive analytics via our lightweight JQuery based frontend.

What's new in 2.2

  • completely rewritten frontend based on backbone.js
  • client-side caching of cubes and dimensions
  • various interaction improvements
  • super easy translation process
  • telemetry for error reports
  • oh yeah - CHARTS!

What didn't make the cut

  • drilling
  • role-based security
  • tablet support
  • embedded mode

These will hopefully make it into 2.3

Backbone.js

  • drastically reduces barrier to entry for contributers
  • makes our lives easier
  • loads and runs even faster (like that were possible)
  • opens up the possibility for alternative backends (XML/A, ElasticSearch, etc)

Caching and performance

  • cubes are now loaded in localStorage, drastically reducing startup time for additional tabs
  • dimensions are prefetched, making new queries instantaneous
  • Backbone allows us to do background rendering of HTML
  • tabs are now stored more efficiently, reducing memory footprint

Translation

  • internationalization is done on the fly
  • users can click a button, see the current translations, and offer new ones
  • translations are sent to CouchDB, where we can pull the data in a format ready to insert into the code base

Telemetry

  • stream errors from every client anonymously
  • like firebug for every browser on the market
  • allows us to fix bugs and immediately relase the fixes
  • it may change the way we distribute Saiku

Charts

  • the most requested feature ever
  • simple charting allows you to pull meaning quickly from the data
  • you can stay in chart mode and continue to massage the data
